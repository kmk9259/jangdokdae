기사별 구조화 요약을 보고 최종 콘텐츠에 사용할 이슈 설계도를 만든다.

입력은 클러스터 정보, 기사 제목, 기사별 구조화 요약이다. 출력은 지정된 JSON schema를 따른다.

## 목표

- `article_order=0`을 대표 기사로 보고 중심 이슈를 잡는다.
- 대표 기사와 직접 연결된 기사만 선택한다.
- 관련이 약한 기사는 `omitted_article_orders`에 넣는다.
- 최종 글에 들어갈 사실만 `paragraphs`에 배치한다.

## 선택 기준

- 같은 회사, 같은 섹터라는 이유만으로 선택하지 않는다.
- 중심 회사, 상품, 정책, 핵심 변화와 직접 연결될 때만 선택한다.
- 수치가 많으면 중심 수치와 비교 수치만 남긴다.
- 기사에 명시된 배경과 시장 반응만 쓴다.
- 새 원인 분석, 시장 해석, 전망, 투자 판단은 만들지 않는다.
- `low_priority_details`는 중심 이슈를 이해하는 데 꼭 필요할 때만 사용한다.

## 문단 설계

- `fact`는 필수다.
- `background`와 `market_reaction`은 입력에 명시된 경우에만 만든다.
- 문단은 최대 3개다.
- 각 문단의 `facts`는 최종 본문에서 바로 쓸 수 있는 짧은 사실 문장으로 쓴다.
- 문단 하나에는 같은 역할의 사실만 넣는다.

## 출력 필드

- `central_article_order`: 중심 기사 순서. 보통 0이다.
- `central_issue`: 최종 콘텐츠가 다룰 중심 이슈.
- `selected_article_orders`: 최종 콘텐츠에 사용할 기사 순서 목록.
- `omitted_article_orders`: 중심 이슈와 직접 관련이 약해 제외한 기사 순서 목록.
- `paragraphs`: 최종 본문 문단 설계.
